What is jQ Skootch?

The JQuery Skootch plugin allows you to easily animate items into the viewport and will shift the already visible DOM items appropriately.

(Click on the 'Contents' Tab to the left for a working example.)

Overview

The plugin provides a method called skootch which is invoked on a trigger container.

Two other containers are required for this plugin to function. An indigen and an invader.

The trigger is the container to which our custom click handler will be bound which triggers the skootch animations.

The invader is the container which will animate into our viewport.

The indigen will be shifted/skootched out of the way to make room for the invader. This container is often the page's primary content.

Basic Usage

Assuming your HTML has the following structure:

<body>
  <div id="skootch-trigger">
    The trigger element
  </div>

  <div id="skootch-invader">
    <p>Some content</p>
  </div>

  <div id="skootch-indigen">
    The "primary" content for the page
  </div>
</body>

Then you can invoke skootch with the following code.

$("#skootch-trigger").skootch();

Let's Talk CSS

jQ Skootch is, at it's core, a wrapper for various animations. With that in mind, there a few CSS rules that we want to have in place.

If we consider the above example, a good starting point for our CSS would be

#skootch-trigger{ 
  height: 31px;
  left: 0px;
  position: absolute;
  width: 48px;
}
#skootch-invader {
  display: block;
  float: left;
  left: -320px;
  position: relative;
  width: 320px;
}
#skootch-indigen {	
  padding: 20px;
  width: 800px;
  margin: 0 auto;
}

The above sets up our invader to be off screen by setting it's relative position to negative it's actual width.

Defining the width of the indigen container is not explicitly required.

The trigger container can pretty much go where ever you like, however, for impact, it is recommended to enforce an absolute or fixed positioning.

Methods

Init:

$('trigger-selector').skootch('op_key', 'op_value');

OR

$('trigger-selector').skootch(settings:Object);

Initializes skootch on the selected element(s) with the passed settings.

See 'Overriding default params' section below for more info on overriding the default params.

Advance:

$('trigger-selector').skootch('advance', callback);

Advances a skootch that has been previously initialized, then executes an optional callback fn.

Retreat:

$('trigger-selector').skootch('retreat', callback);

Retreats a skootch that has been previously initialized, then executes an optional callback fn.

Destroy:

$('trigger-selector').skootch('destroy', callback);

Removes all skootch elements for selected item(s) from the DOM and returns non-dynamic items to their original state, then executes an optional callback fn.

Properties

$.fn.skootch.defaults = {
  advanceEasing:        'swing',
  advanceSpeed:         'slow',
  indigen:              '#skootch-indigen',
  indigenUseMargins:    false, 
  invader:              '#skootch-invader',
  invaderClickCallback: null,
  invaderClickRetreat:  true,
  invaderLinks:         '#skootch-invader a, 
			 #skootch-invader input[type=submit]',
  justify:              'left',
  minInvaderMargin:     40,
  retreatEasing:        'swing',
  retreatSpeed:         'slow',
  smart:                true,
  squashOverflow:       'true',
  triggerClosed:        'skootch-trigger-closed',
  triggerOpen:          'skootch-trigger-open',
  wrapperSuffix:        'skootch-wrap'
};

advanceEasing:String = "easing fn name"
default = "swing"
advancing easing function.

advanceSpeed:String = "easing speed"
default = "slow"
advancing animation speed.

indigen:String = "id-or-classname"
default = "#skootch-indigen"
the id or class name used for the element that will be skootched.

indigenUseMargins:Boolean
default = false
should we use Margins to calculate the total width of the $indigen elem (the container that skootch is invoked upon).

invader:String = "id-or-classname"
default = "#skootch-invader"
the id or class name used element that will skootch into the window.

invaderClickCallback:Function
default = null
callback for the invaderLinks on click.

invaderClickRetreat:Boolean
default = true
should everything skootch back to it's start position if a invaderlink is clicked?

invaderLinks:String = "id-or-classnames"
default = "#skootch-invader a, #skootch-invader input[type=submit]"
if there are links in the invader elem these are them.

justify:String = "direction"
default = "left"
skootch trigger justification, currently only 'right' or 'left' are supported.

minInvaderMargin:Number = "num in pixels"
default = 40
the minimum amount of margin applied to the invader elem - ONLY USED IF smart = true.

retreatEasing:String = "easing fn name"
default = "swing"
retreat easing function.

retreatSpeed:String = "easing speed"
default = "slow"
retreat animation speed.

smart:Boolean
default = true
should we change the amount we animate our skootched elems based on window size?

squashOverflow:Boolean
default = true
should overflow-x be present when the invader is in view?

triggerClosed:String = "classname-without-."
default = "skootch-trigger-closed"
trigger closed status class.

triggerOpen:String = "classname-without-."
default = "skootch-trigger-open"
trigger open status class.

wrapperSuffix:String = "suffix-added-to-wrapper"
default = "skootch-wrap"
the div that will wrap our DOM node that will be skootched by our 'invader'.

Overriding default params

The default options can be overridden by passing a property name and value as separate parameters...

$("#div").skootch("justify", "right");

...or by passing a single map of key-value pairs to the skootch method.

$("#div").skootch({
                    trigger: "#skootch-trigger", 
                    invader: "#skootch-invader", 
                    invaderlinks: "a.invaderlinks", 
                    advancespeed: "fast", 
                    retreatspeed: 1000
                });
